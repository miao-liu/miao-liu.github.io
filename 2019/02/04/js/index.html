重排(Reflow)

  1. 添加会删除可见的dom元素
  2. 元素的位置改变
  3. 元素尺寸改变(因编剧，填充功能，边框宽度，高度等属性改变时)
  4. 内容改变(文字改编成图片被另一个不同尺寸的所替代)
  5. 最初的页面渲染
  6. 浏览器窗口改变尺寸

  • 根据改变的性质，渲染树上或大或小的一部分需要重新计算。
    某些改变可导致重排版整个页面，例如一个滚动条出现时获取局部信息的操作将导致刷新队列动作，这意味着使用了下面的这些方法offsetTop,offsetLeft,offsetWidth,offsetHeight,scrollTop,scrollLeft,scrollWidth,scrollHeight, clientTop,clientLeft,clientWidth,clientHeight,getComputedStyle(currentStyle in IE)操作这些属性都将会引起重排

  • 减少重排版的方法：

  • 动画
    使用绝对左边定位页面动画的元素，使他位于页面布局流之外
    启动元素动画，当他扩大时，覆盖部分页面，这是一个重回过程，但只影响页面的小部分，避免重排版并重绘一大块页面
    当动画结束时，重新定位，从而只一次下移文档其他元素的位置
  • 元素
    通过改变display属性，临时从文档上溢出元素然后再恢复它
  • 创建一个文档片断
    它是一个轻量级的domcument对象，可对其实施操作，document.createDocumentFragment()创建要更新节点的副本，在其副本上操作最后覆盖老节点el.cloneNode(true)

算法和流程控制

  • 基于循环迭代:for,while.do-while,for-in(slow)
  • 基于函数迭代:forEach(slower）
  • 表达式:if-else,switch(可读性好，且快)
  • 减少迭代次数:Duffs Devices(达夫设备)
  • 查表法
  • 当使用查表法是，必须完全消除所有条件判断。操作转换成一个数组项查询或者一个对象成员查询，它的优点是，由于没有条件判断，当后选址数量增加时，很少，甚至没有增加额外的性能开销
  • 递归
  • 最大的问题时，一个错误定义，挥着缺少终结条件可导致长时间运行，冻结用户界面，此外，低估函数还会遇到浏览器调用栈大小的限制

Web Workers网页工人线程的运行环境
